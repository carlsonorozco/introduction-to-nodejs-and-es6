What is NodeJS

NodeJS is an open-source JS runtime environment built on Google's V8 engine.

Myths and truths about Node.js

Myths:

  • Node.js is a framework
  • Relies on Chrome V8 engine only
  • It does not support multi-threads
  • Beginner developers can’t work with Node

Actual facts:

  • Node.js is basically a server capable of executing JavaScript
  • It is an open source and cross-platform system for building web applications
  • It does, providing asynchronous and event-driven APIs
  • Any programmer familiar with JavaScript can learn Node quickly

What are the key features of NodeJS?

NodeJS uses an event-driven IO model and is designed to scale through non-blocking calls in a single threaded environment. NodeJS comes with a standard library which includes the most common utils for web and backend programming. NodeJS also comes bundled with NPM package manager.

What is Node REPL?

REPL stands for Read Eval Print Loop and it represents a computer environment like a Windows console or Unix/Linux shell where a command is entered and the system responds with an output in an interactive mode.

Node.js or Node comes bundled with a REPL environment. It performs the following tasks:

  • Read − Reads user's input, parses the input into JavaScript data-structure, and stores in memory.
  • Eval − Takes and evaluates the data structure.
  • Print − Prints the result.
  • Loop − Loops the above command until the user presses ctrl-c twice.

What is NPM?

NPM is the default package manager for NodeJS. At present, it claims to be the default package manager for the entire JS ecosystem.

What are the key features of NPM?

NPM holds the largest registry of open-source libraries in the world. The registry has no vetting process, so anyone can publish a package of any quality. NPM is written in JavaScript and consists of a CLI client and remote registy powered by CouchDB.

What is ECMAScript (ES)?

A language standardized by ECMA International and overseen by the TC39 committee. This term is usually used to refer to the standard itself.

What is ECMAScript 5 (ES5)?

The 5th edition of ECMAScript, standardized in 2009. This standard has been implemented fairly completely in all modern browsers

What is ECMAScript 6 (ES6)/ ECMAScript 2015 (ES2015)?

The 6th edition of ECMAScript, standardized in 2015. This standard has been partially implemented in most modern browsers. To see the state of implementation by different browsers and tools, check out these compatibility tables.

What is ECMAScript 2016?

The expected 7th edition of ECMAScript. This is scheduled to be released next summer. The details of what the spec will contain have not been finalized yet

What is ECMAScript Proposals?

Proposed features or syntax that are being considered for future versions of the ECMAScript standard. These move through a process of five stages: Strawman, Proposal, Draft, Candidate and Finished.
